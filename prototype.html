Prototype Walkthrough

A guided tour of the Sakay Partido mobile application screens and key features.

📱

About the Prototype

Sakay Partido is a mobile-first application built to connect commuters at Partido State University with nearby motorcycle (door-to-door) and tricycle drivers. The prototype was developed for usability testing and covers the core booking flow from login to ride completion.

Key Features

🔐

Secure Login

Phone number and password authentication with a "Remember Me" option for convenience.

🛵

Ride Type Selection

Choose between motorcycle (door-to-door) or tricycle services based on your needs.

📍

Route Finder

Set your pickup and drop-off points easily within the Partido State University area.

⚡

Quick Booking

Book a ride in just a few taps with a streamlined, minimal-step interface.

📋

Ride Summary

Clear confirmation screen showing driver info, route, and estimated fare before confirming.

👤

User Profile

Manage personal information and view ride preferences from your profile page.

Screen-by-Screen Walkthrough

1

Splash / Welcome Screen

The app opens with the Sakay Partido logo and a brief animation before directing users to the login screen. The splash screen reinforces brand identity with the blue color palette.

2

Login Screen

Users enter their registered phone number and password. A "Remember Me" toggle and "Forgot Password" link are provided for convenience. New users can tap "Sign Up" at the bottom. The login screen is illustrated in the prototype screenshot below.

Login Screen — Sakay Partido Prototype

3

Registration / Sign Up

New users provide their name, phone number, and create a password. Input validation ensures all fields meet requirements before allowing registration. A confirmation message is shown upon successful account creation.

4

Home / Dashboard

After login, users land on the main dashboard which displays available ride types — Motorcycle and Tricycle. Quick-access buttons let users immediately start a booking. The dashboard also shows recent ride history if available.

5

Ride Type Selection

Users choose between Motorcycle (door-to-door) or Tricycle. Each option displays an icon, brief description, and estimated availability in the area. This step ensures users pick the right transport mode for their needs.

6

Set Pickup & Drop-off Location

Users type or select their pickup and destination points. The interface is designed for the Partido State University area, with common locations pre-suggested for faster input.

7

Booking Confirmation

A summary screen shows the selected ride type, pickup, destination, and estimated fare. Users confirm the booking by tapping the "Book Now" button. A confirmation dialog appears to prevent accidental bookings.

8

Waiting for Driver

After booking, users see a status screen indicating that the system is searching for an available driver. A cancel option is available in case the user changes their mind.

9

Driver Matched / Ride Details

Once a driver is matched, the app displays driver name, contact number, vehicle description, and estimated arrival time. Users can call or message the driver directly from this screen.

10

Ride Completion

After the ride, the app shows a completion screen with a ride summary and a prompt to rate the experience. This closes the feedback loop and helps maintain service quality.

Design Notes

🎨 Visual & UX Design Decisions

The Sakay Partido prototype was designed with a clean, minimal blue-dominant color scheme that conveys trust, reliability, and movement — all qualities essential for a transportation service. The use of rounded corners, card-based layouts, and clear call-to-action buttons follows current mobile UI best practices.

Typography uses a bold, readable font hierarchy to guide user attention naturally from headings to actions. White space is used intentionally to reduce cognitive load, particularly on the booking and confirmation screens where clarity is critical.

Every screen was designed with the target audience in mind — young university students who are comfortable with smartphones but may be first-time users of a dedicated ride-booking platform. Simplicity and speed were the guiding principles throughout the design process.
